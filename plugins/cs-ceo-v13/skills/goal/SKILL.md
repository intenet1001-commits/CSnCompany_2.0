# Goal Gate — 목표 추출 + 명확화

## 역할

요청에서 "무엇을 달성하고 싶은가"를 추출한다.
HOW(방법/도구)가 아니라 WHAT(목표/결과)를 파악하는 것이 유일한 목적이다.
CEO는 이 스킬을 모든 요청에서 Phase -3 이전에 가장 먼저 실행한다.

---

## 프로토콜

### STEP 1: 목표 신호 분석

요청을 분석해 아래 기준으로 명확도를 판단한다.

**명확 — 질문 생략, 즉시 반환:**

| 패턴 | 예시 | 추출 목표 |
|------|------|---------|
| URL + 동사 | "이 URL 테스트해줘" | URL의 버그/성능/보안 문제 탐지 |
| 도메인 + 기능 | "결제 기능 설계해줘" | 결제 기능의 설계 문서 산출 |
| 문제 + 기대결과 | "뭔가 이상한 것 같아" | 코드베이스 이상 원인 파악 |
| 구체적 작업 | "전체 코드 리뷰해줘" | 코드 품질·보안·성능 리포트 |
| with 파트너 포함 | "with gstack: 결과 정리해줘" | 분석 결과를 구글 시트에 저장 |

**불명확 — 질문 필요:**

| 패턴 | 예시 | 이유 |
|------|------|------|
| 동사만 있음 | "개선해줘", "봐줘" | 무엇을 개선? 어떤 기준으로? |
| 맥락 없는 키워드 | 파일명만, 에러코드만 | 이 파일/에러로 무엇을 하길 원하는가? |
| 범위가 무한정 열림 | "뭐든 다 봐줘", "전체 다 해줘" | 완료 기준 불명확 |
| 목표와 방법이 혼재 | "gstack으로 해줘" | 무엇을 gstack으로? |

---

### STEP 2: 불명확 시 1회 질문

```
AskUserQuestion(
  question: "어떤 목표를 달성하고 싶으신가요?\n현재 요청: '[원문 요청]'",
  options: [
    "현재 요청 그대로 진행 (목표 생략)",
    "작업 취소"
  ]
)
```

- **Other(직접 입력)**: 입력한 텍스트를 goal_statement로 사용 → STEP 3
- **"현재 요청 그대로 진행"**: 원문을 goal_statement로 사용 → STEP 3 (was_clarified: false)
- **"작업 취소"**: CEO 즉시 종료

---

### STEP 3: 구조화 목표 확정

```
GOAL = {
  goal_statement:    "한 문장으로 정제된 목표 (WHAT 중심, HOW 제외)",
  scope:             "영향 범위 — 특정 파일 / 특정 기능 / 전체 코드베이스",
  success_criteria:  "완료 판정 기준 1-2줄 (무엇이 있으면 완료인가)",
  was_clarified:     true (사용자가 목표를 명시했을 때) / false (원문 그대로)
}
```

이 GOAL 객체가 Phase G의 반환값이며, CEO의 Phase -3 이후 전체 흐름에 사용된다.

---

## 예시

**입력**: "이 URL 테스트해줘 https://example.com"
```json
{
  "goal_statement": "example.com의 버그·성능·접근성 문제를 탐지하고 개선점 리포트 산출",
  "scope": "example.com 전체 웹 페이지",
  "success_criteria": "14-point 테스트 완료 리포트 + 우선순위별 수정 목록",
  "was_clarified": false
}
```

**입력**: "개선해줘" (불명확 → 질문 후 사용자가 "UI 버튼 반응속도 개선"이라 답변)
```json
{
  "goal_statement": "UI 버튼 클릭 반응속도를 개선하여 사용자 경험 향상",
  "scope": "UI 버튼 컴포넌트 및 관련 이벤트 핸들러",
  "success_criteria": "버튼 클릭 → 응답 지연 < 100ms, 시각적 피드백 즉시",
  "was_clarified": true
}
```
